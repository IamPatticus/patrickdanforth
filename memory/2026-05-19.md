- Root cause: wrong DT overlay. The Sunfounder 3.5" is an IPS variant that needs `mhs35ips` not `mhs35`.
- Standard `mhs35` overlay has init sequence for TFT panels — wrong gamma, wrong color order, wrong everything for IPS.
- Patticus used Perplexity to find the fix: cloned SunFounder LCD-show repo, ran `sudo ./MHS35IPS-show`.
- Script installed `/boot/overlays/mhs35ips.dtbo` and updated config.txt to `dtoverlay=mhs35ips:rotate=90`.
- Colors now correct. IPS overlay has completely different init sequence (gamma, power, register ordering).
- Source: https://github.com/sunfounder/LCD-show.git
- SunFounder warns newer kernels can break the IPS driver — something to watch on upgrades.
- Patticus says screen photo shows ~3 colors (red, black, blue, orange), source image is normal
- Pushed pure red (255,0,0) test frame to screen — awaiting Patticus to confirm what color appears
- If blue shows instead of red → confirmed R↔B channel swap issue in framebuffer
