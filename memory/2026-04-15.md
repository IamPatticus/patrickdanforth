## Brain App Fixes (2026-04-15)

### Bug 1: Missing Drive scope
`SheetsSyncManager` was only requesting `SPREADSHEETS` scope, but `spreadsheets().list()` to find the existing "Headache Log" spreadsheet requires `DRIVE_API` scope. Added:
- `Scope("https://www.googleapis.com/auth/drive")` to GoogleSignInOptions
- Same Drive scope added to `GoogleAccountCredential` in `buildSheetsService()`

### Bug 2: onResume() was logging user out
`MainActivity.onResume()` called `handleSignInResult(null)` which throws `ApiException` and sets `isSignedIn = false` on **every app resume** — meaning the user could never stay signed in. Fixed by replacing with a proper `refreshSignInState()` method that reads `GoogleSignIn.getLastSignedInAccount()` without triggering the broken intent flow.

Files changed:
- `data/SheetsSyncManager.kt` — added Drive scope (×2) + new `refreshSignInState()` method
- `ui/MainActivity.kt` — `onResume()` now calls `refreshSignInState()` instead

### Brain App Sheets Sync — Fixed and verified ✅
Three bugs were preventing Google Sheets sync from working:

1. **Missing Drive scope** (`SheetsSyncManager.kt`) — `spreadsheets().list()` requires Drive API access to find existing spreadsheets. Added Drive scope to both GoogleSignInOptions and GoogleAccountCredential.

2. **Missing Drive scope in sign-in intent** (`HeadacheViewModel.kt`) — The ViewModel's own GoogleSignInClient was also only requesting SPREADSHEETS scope, so even fresh sign-ins never got Drive access. Added Drive scope there too.

3. **`onResume()` logout bug** (`MainActivity.kt`) — Calling `handleSignInResult(null)` on every resume threw an ApiException and set `isSignedIn = false`, logging the user out every time they switched away from the app. Replaced with proper `refreshSignInState()` method.

Also improved error messages so failures no longer show as "sync failed: null."

Fixed and confirmed working by Patticus on 2026-04-15.
