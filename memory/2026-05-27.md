### What was killed

### What now runs on Nexus
**`nexus-dashboard.py`** — systemd service, runs as root (GPIO access), enabled at boot.

Features:
- **Solar Dashboard** — pulls from `http://192.168.1.103:18991/solar-snapshot.json` every 30s
- **Two-way messaging with Talos** — background thread polls inbox every 5s
- Messages from Talos appear as an overlay banner on the Whisplay (8-second display)
- **Whisplay button** wired — pressing it sends a message back to Talos via POST to outbox
- Pisugar server still running for battery monitoring (ports 8421-8423)

### Talos-side infrastructure
**`services/mission-control/server.py`** — HTTP server on port 18991 (`0.0.0.0`), systemd service.

Endpoints:
- `GET /solar-snapshot.json` — static solar data
- `GET /nexus-inbox.json` — messages from Talos → Nexus
- `POST /nexus-outbox.json` — messages from Nexus → Talos
- `GET /nexus-send?text=...&from=...` — send message TO Nexus (CLI/debug)

**`scripts/nexus_relay.py`** — Python helper for sending/receiving programmatically.

### Two-way relay architecture
```
Talos (Pi 5)                          Nexus (Pi Zero 2 W)
┌─────────────────────┐              ┌──────────────────┐
│ mission-control     │──HTTP GET──→ │ nexus-dashboard  │
│ server.py :18991    │←─HTTP POST── │ .py              │
│                     │              │ Whisplay 240x280  │
│ /nexus-inbox.json   │  messages    │ Button → POST msg │
│ /nexus-outbox.json  │  both ways   │ Inbox poll 5s     │
│ /nexus-send?text=   │              │ Solar poll 30s    │
└─────────────────────┘              └──────────────────┘
```

### Why no OpenClaw on Nexus
OpenClaw uses ARM64 native addons (canvas, pty, clipboard) with `linux-arm64` builds only. Pi Zero 2 W is ARMv7 (32-bit) — no compatible builds exist. The HTTP relay approach works for pure messaging without the overhead.
