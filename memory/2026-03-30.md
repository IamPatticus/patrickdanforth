## Mission Control Routing Fixed!

Mission Control (`/mission/`) is back live at `https://serenity.tail4695cd.ts.net/mission/`.

**What was broken:**
- Tailscale Serve was routing `/` directly to OpenClaw gateway (18789) instead of a central reverse proxy
- `/mission` had a redirect loop file (`mission` plain file) pointing to `/mission/` which 404'd on the static server
- OpenClaw gateway was catching all routes and serving its Control UI instead of static files

**What was fixed:**
1. Created `reverse_proxy.py` on port 8080 as the central router for all Tailscale Serve traffic
2. Routes: `/whoop/*` → Flask (5001), `/mission/*` → mission-control static files, `/` → OpenClaw gateway
3. Changed `<base href>` in all mission-control HTML files back to `/mission/` so relative links work
4. Added special handling for `/sanctum.html` and `/devices.html` at root URL (since they use `base href="/mission/"`)
5. Added redirect from `/mission` → `/mission/`
6. Reset Tailscale Serve to point to reverse proxy on 8080 instead of gateway directly
7. Created `reverse-proxy.service` systemd user unit — survives reboots
8. MPPT Watch panel removed (devices not publishing data locally)

**Files changed:**
- `services/reverse_proxy.py` — new central router
- `services/mission-control/index.html` — base href restored to `/mission/`
- `services/mission-control/sanctum.html` — base href restored to `/mission/`
- `services/mission-control/devices.html` — base href restored to `/mission/`
- Removed `services/mission-control/mission` redirect loop file
