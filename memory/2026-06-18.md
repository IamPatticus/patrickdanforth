# 2026-06-18

## Mission Control / Tailscale Fix (Morning)

**Problem:** `https://serenity.tail4695cd.ts.net/mission/` was loading the OpenClaw Control UI instead of Mission Control.

**Root cause:** Two things fighting each other:
- OpenClaw's built-in Tailscale integration (`gateway.tailscale.mode: "serve"`) managed `tailscale serve` pointing at the gateway port (18789)
- Mission Control proxy (`mission_proxy.py`) was running on port 18888 and routing `/mission/` → dashboard, but Tailscale never hit it

**Architecture before:**
```
Tailscale serve → port 18789 → OpenClaw Gateway → serves Control UI for all paths
                 Mission proxy on 18888 (orphaned, never reached)
```

**Architecture after:**
```
Tailscale serve → port 18789 → Mission Proxy → /mission/ → dashboard files
                                           → /*      → Control UI (port 18790)
                 port 18790 → OpenClaw Gateway
```

**Changes made:**
1. **Mission proxy** updated to listen on **18789**, forward to gateway on **18790**
2. **Gateway port** changed from 18789 → **18790** in `openclaw.json` AND in `systemd` service file
3. **OpenClaw Tailscale** disabled (`gateway.tailscale.mode: "off"`) — we manage `tailscale serve` manually now
4. **Systemd service** `mission-control-proxy.service` created — auto-starts on boot, waits for gateway
5. **Tailscale serve** manually pointed at port 18789

**Verification:**
- `/` → Control UI ✅
- `/mission/` → Mission Control ✅
- `/mission` → 301 → `/mission/` ✅
- `systemctl --user status mission-control-proxy` → active ✅
