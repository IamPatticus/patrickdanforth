# 2026-03-28

- Spent a long debugging session getting `pizero-openclaw` working on the Raspberry Pi Zero 2 W with WhisPlay hardware.
- Confirmed the main blocker was network configuration plus Pi-side runtime setup, not the OpenClaw gateway itself.
- For remote use, the correct client route is through Tailscale Serve using `https://talos.tail4695cd.ts.net`, not raw LAN or tailnet IP + port when the gateway is loopback-bound behind Serve.
- Pi Zero needed Tailscale installed/enabled for remote access to the OpenClaw gateway.
- Working client environment on the Pi Zero uses a project-local venv named `venv` (not `.venv`) and a project-root `.env` file loaded before launch.
- During troubleshooting, direct HTTP to `100.64.93.50:18789` timed out because the gateway was loopback-only while using Tailscale Serve; switching back to the Serve URL fixed connectivity.
- Remaining hardware quirk: WhisPlay button init on the Pi Zero still hit `RuntimeError: Failed to add edge detection` in `WhisPlay.py`, but the device otherwise reached the gateway and worked once network/auth were corrected.
