## Wear OS Tile — full implementation built
**Architecture:** Phone app ↔ Data Layer ↔ Watch Tile (no standalone watch app needed)

### Phone-side (app/src/main/.../wear/):
- `WearDataLayer.kt` — Singleton that observes Room DB via Flow, auto-pushes HeadacheState to watch on any change, handles start/stop commands from watch
- `WearListenerService.kt` — Routes watch messages to WearDataLayer
- `HeadacheApplication.kt` — Custom Application class that inits WearDataLayer on startup

### Watch-side (wear/ module):
- `tile/HeadacheTileService.kt` — ProtoLayout tile showing active headache timer/type/pain or "Start" button when idle
- `data/DataLayerSync.kt` — Singleton holding current HeadacheState, sends start/stop messages to phone
- `data/DataLayerSyncService.kt` — WearableListenerService receiving state from phone
- Brain icon drawables copied from phone module
