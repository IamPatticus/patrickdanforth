### What We Did Today

#### Tile Installation
- Built `wear-debug.apk` from the `:wear` module
- ADB serials kept changing (the `(2)`, `(3)` suffixes) due to ADB re-registering the watch
- Used Android Studio's own ADB (`$env:LOCALAPPDATA\Android\Sdk\platform-tools\adb.exe`) to avoid version mismatch
- Successfully installed wear APK to watch

#### Bugs Discovered & Fixed

1. **Phone START button broken after watch APK install**
   - Root cause: `WearDataLayer.kt` handlers for `START_PATH` and `STOP_PATH` were using `repository.getAllEpisodes().collectLatest` inside coroutines that never canceled — leaking coroutine scopes and blocking subsequent start calls
   - Fix: Replaced `collectLatest` with `first()` to read current value once and cancel immediately
   - Phone app working again after force stop + reinstall with fix

2. **Watch DataLayer never connecting**
   - Root cause: `DataLayerSyncService` (extends `WearableListenerService`) only gets bound by Google Play Services when data/message events arrive. Since no events fire before connection, the service never starts and `DataLayerSync.init()` is never called
   - Fix attempted: Added `DataLayerSync.init()` call inside `HeadacheTileService.onTileRequest()` so the tile itself discovers the phone node when it renders
   - Status: **NOT YET VERIFIED** — the rebuilt wear APK with this fix may not have been successfully installed (no logcat output from the watch)

### Current State
